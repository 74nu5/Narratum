Direction artistique

Une refonte éditoriale

On abandonne le langage Fluent au profit d'une esthétique « livre » chaleureuse : titres et prose en Newsreader (serif), interface en Hanken Grotesk (sans), accent sienna, papier crème, et un thème nuit assorti. Ce document est la source de vérité de la migration : tokens, composants réutilisables, blueprints des 5 écrans et correspondances Fluent → custom. Basculez le thème (☾/☀) en haut à droite pour vérifier les deux modes.

Maquettes de référence (pixel-près, interactives) dans reviews/Révision IHM générateur d'histoire/ — Accueil, Bibliotheque, Creer, Lecture, Narratum. Ce guide en extrait le système et le fige en classes CSS.

4 · Notes de migration (Fluent → custom)

On retire Microsoft.FluentUI.AspNetCore.Components et on rend en HTML natif + cette feuille de style.

Correspondances composants

Fluent v5 (actuel)RemplacementClasse
FluentButton (ButtonAppearance)<button> / <a>.btn .btn-accent|ghost|soft|dashed|danger|quiet
FluentSelect (générique TValue)<select> natif.field + .fld-label
FluentTextInput / FluentTextArea<input> / <textarea>.field
FluentText (Size/Weight)<h1..h3> serif, <p>, .kicker, .eyebrow—
FluentCard<div>.card
FluentSpinnerSVG/CSS accent spinner maison(à créer)
FluentMessageBar<div> ton statut.note (variante danger/success)
FluentLayout / providers.appbar + <main> + .appfoot—
DialogService.ShowConfirmationAsyncModale maison (overlay + carte)voir §2
Timeline (pastilles par page)Barre de progression + ◀ ▶ + « Aller à ».progress

Changements structurels

  • Thème jour/nuit : classe .night sur l'élément racine, togglée par le bouton ☾/☀ ; persister le choix (localStorage via JS interop, ou service scоped + cookie).
  • Polices Newsreader + Hanken Grotesk : <link> Google Fonts dans App.razor, ou self-host dans wwwroot pour l'offline.
  • theme.css unique (tokens + classes ci-dessus) chargé globalement ; retirer le bundle Fluent et les --colorNeutral*.
  • Layout génération : sidebar bibliothèque persistante à gauche (270px, sticky) + colonne centrale (max 1020px) ; le composer « Écrire la suite » est une carte étroite à droite de la rangée de choix.
  • Prose : lettrine sur le premier paragraphe, serif 18–19px, interligne 1.72–1.78.
  • Mode Expert : les coulisses des agents passent en modale déclenchée par un bouton pointillé, plus un panneau inline.
  • En-tête global : logo N + Narratum, toggle Expert (pill), toggle thème, réglages ; sur la page Lecture, pilule centrale « 📖 Lecture · <titre> ».
  • Réglages (⚙) : la page /config reste, à re-styler avec .field/.card (non maquettée — reprendre le langage).

Ordre suggéré

  • 1 · Poser theme.css (tokens + classes) + polices + toggle jour/nuit dans le layout.
  • 2 · Reconstruire l'en-tête / layout global (appbar + footer) sans Fluent.
  • 3 · Écran par écran : Accueil → Bibliothèque → Créer → Lecture → Narratum (le plus riche en dernier).
  • 4 · Modales (suppression, coulisses) + états vides + spinner de génération.
  • 5 · Retirer les PackageReference Fluent du .csproj une fois le dernier composant migré.
